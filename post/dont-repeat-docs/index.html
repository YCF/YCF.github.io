文档也不应该重复造轮子

2020-01-12

事情起端是这样的，准备写个网盘小工具，开源到Github，想写个高大上的 README，因为太懒所以就找模板，自然就有了这个文章。

写程序很多人都说是不要重复造轮子，就是说先找找别人写的，那为啥文档不是呢？至少格式应该用一个模板，站在巨人肩膀上的感觉真的很棒。

文档格式，开源协议，这都是隐形的造轮子重灾区呢！

几个有用的链接：

  • README的艺术
  • Github readme 模板
  • 参与者公约
